# Winner Partner 用户手册

## 📋 项目概述

**Winner Partner** 是一个基于像素检测的围棋辅助工具，能够实时扫描屏幕上的深绿色棋子位置，并将其转换为标准围棋坐标。该程序专为在线围棋平台设计，通过颜色识别技术实现自动化棋盘状态检测。

---

## ✨ 核心功能

### 1. 智能像素检测
- **深绿色识别**: 精准检测棋盘上的深绿色棋子 (HSV 颜色空间)
- **短扫算法**: 在 361 个交叉点进行局部扫描，避免全局处理
- **防重复机制**: 自动记录已扫描位置，避免重复检测
- **二次验证**: 向上逐像素检测，区分白色/黑色背景

### 2. 坐标转换系统
- **屏幕 → 围棋**: 将像素坐标 (x, y) 转换为围棋坐标 (如 Q16)
- **围棋 → 屏幕**: 反向转换，支持手动输入坐标定位
- **自适应映射**: 根据红色矩形区域动态计算网格

### 3. 交互式界面
- **区域选择器**: 可视化拖拽选择监控区域
- **实时日志**: 记录所有操作和检测结果
- **主题切换**: 支持浅色/深色模式
- **窗口置顶**: 可选始终显示在最上层

### 4. 数据持久化
- **配置保存**: 自动保存监控区域、主题等设置
- **日志导出**: 结束程序时将日志保存到 `mygame.txt`
- **断点续扫**: 重启后保留已扫描位置记录

## 🎮 使用说明

### 第一步：启动程序

```bash
# 方法 1: 直接运行
python src/main.py

# 方法 2: 使用启动脚本
./winnerpartner.sh
```

### 第二步：选择监控区域

1. 点击 **"重新确定位置"** 按钮
2. 屏幕变暗，出现十字光标
3. **拖动鼠标** 绘制红色矩形框住整个棋盘
4. **松开鼠标** 确认区域
5. 可拖动矩形调整位置，或拖动角标调整大小

**提示**: 确保矩形包含完整的 19×19 棋盘网格

### 第三步：确认区域

1. 点击 **"确定"** 按钮
2. 系统会清空已扫描位置记录
3. 状态栏显示 **"矩形区域已确认"**

### 第四步：扫描棋子

1. 点击 **"扫描"** 按钮
2. 程序自动检测 361 个交叉点
3. 检测到深绿色棋子后：
   - 转换为围棋坐标 (如 `Q16`)
   - 显示在 **"文字显示"** 框中
   - 记录到操作日志
   - 标记该位置为已扫描

**扫描逻辑**:
```
对于每个交叉点 (x, y):
  1. 提取周围 10×10 像素区域
  2. 统计白色/黑色/深绿色像素数量
  3. 如果 白+黑 > 300 → 跳过 (无棋子)
  4. 如果 深绿 > 300 → 有效 (有棋子)
  5. 标记该位置，下次不再扫描
```

### 第五步：手动下棋 (可选)

1. 在 **"输入坐标"** 框中输入围棋坐标 (如 `a15`)
2. 点击 **"下棋"** 按钮
3. 系统执行鼠标双击操作
4. 记录到日志

**坐标格式**:
- 小写字母 + 数字 (如 `q16`, `a1`, `t19`)
- 不包含字母 `i` (围棋标准)

### 第六步：结束程序

1. 点击 **"结束"** 按钮
2. 日志自动保存到 `mygame.txt`
3. 程序关闭

## 🔍 核心技术原理

### 1. 像素检测算法

#### 颜色范围定义

```python
# 白色范围 (RGB)
lower_white = [200, 200, 200]
upper_white = [255, 255, 255]

# 黑色范围 (RGB)
lower_black = [0, 0, 0]
upper_black = [50, 50, 50]

# 深绿色范围 (RGB)
lower_green = [0, 100, 0]
upper_green = [50, 255, 50]
```

#### 短扫流程

```
输入: 中心点 (x, y)
输出: "skip" / "valid" / None

1. 提取 (x±10, y±10) 区域
2. 创建颜色掩码 (cv2.inRange)
3. 统计各颜色像素数量
4. 判断:
   - 白+黑 > 300 → 返回 "skip" (空位)
   - 深绿 > 300 → 返回 "valid" (有子)
   - 其他 → 返回 None (继续观察)
```

### 2. 坐标映射算法

#### 屏幕 → 围棋

```python
def screen_to_board(x, y):
    # 1. 计算网格尺寸
    cell_width = (x2 - x1) / 18
    cell_height = (y2 - y1) / 18
    
    # 2. 计算行列索引
    col_index = round((x - x1) / cell_width)  # 0-18
    row_index = round((y - y1) / cell_height)  # 0-18
    
    # 3. 转换为围棋坐标
    col = 'abcdefghjklmnopqrst'[col_index]
    row = 19 - row_index  # 从上到下 1-19
    
    return f"{col}{row}"
```

#### 围棋 → 屏幕

```python
def board_to_screen(coordinate):
    # 1. 解析坐标
    col_char = coordinate[0]  # 'q'
    row = int(coordinate[1:])  # 16
    
    # 2. 计算索引
    col_index = cols.index(col_char)  # 16
    row_index = 19 - row  # 3
    
    # 3. 计算屏幕坐标
    x = x1 + col_index * cell_width
    y = y1 + row_index * cell_height
    
    return (int(x), int(y))
```

## 🐛 常见问题

### Q1: 无法检测到棋子

**可能原因**:
1. 监控区域未正确设置
2. 棋子颜色不是深绿色
3. 阈值设置不当

**解决方案**:
1. 重新选择监控区域，确保包含完整棋盘
2. 检查游戏平台棋子颜色
3. 降低 `GREEN_THRESHOLD` 到 200

### Q2: 坐标转换不准确

**可能原因**:
1. 监控区域包含边框
2. 棋盘不是标准 19×19
3. 屏幕缩放比例非 100%

**解决方案**:
1. 精确框住棋盘内部网格
2. 确认游戏平台使用标准棋盘
3. macOS: 系统偏好设置 → 显示器 → 缩放 → 默认

### Q3: 重复检测同一位置

**可能原因**:
1. 已扫描位置记录被清空
2. 程序重启

**解决方案**:
1. 正常现象，每次扫描后会标记
2. 如需持续监控，不要频繁重启

### Q4: 区域选择器无法显示

**可能原因**:
1. PyQt5 权限问题
2. 多显示器配置

**解决方案**:
```bash
# macOS: 授予屏幕录制权限
系统偏好设置 → 安全性与隐私 → 隐私 → 屏幕录制
勾选 Terminal 或 Python

# 多显示器: 确保在主显示器运行
```
